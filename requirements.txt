# TensorFlow MUST be pinned explicitly. deepface==0.0.92 only declares a loose
# `tensorflow>=1.9.0`, so on a fresh install pip resolves whatever is newest
# (e.g. 2.21.x) and only *backtracks* to 2.16.2 because tf-keras==2.16.0
# constrains it — a fragile resolution that varies by pip version / platform.
# When pip lands a TF != 2.16.2, the every-launch dependency_health_check
# (REPAIR_PACKAGES expects exactly 2.16.2 + tensorflow-intel==2.16.2 on win32)
# fails and forces a repair cycle on first launch; on a slow/firewalled
# connection that repair can hard-fail and the launcher exits before the GUI.
# Pinning here (mirrors requirements-hashed.txt + REPAIR_PACKAGES) makes the
# first install land the exact stack the health probe expects — no repair loop.
# numpy<2 is required: TF 2.16.2's ml-dtypes~=0.3.1 needs numpy 1.26.x; an
# unpinned numpy 2.x breaks TF import.
