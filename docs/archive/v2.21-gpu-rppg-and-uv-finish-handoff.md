### DELIVERABLE A (PRIMARY, URGENT): Fix the friend's GPU rPPG — it runs on CPU, 20 min/iteration

The friend has an **RTX 4080**, is on the **v2.17-personal** build (the one-time
hand-patched portable zip where we fixed `run_rppg.bat`), and **every rPPG runs
on CPU → ~20 minutes per video** (10 iterations each). His log is saved in the
repo at **`_friend_logs/friend_rtx4080_v217_rppg_cpu.txt`** (gitignored). The
smoking gun (verbatim from his log):

```
[10:06:09] GPU: CuPy ready — rPPG uses GPU acceleration.          <- GUI process: CuPy WORKS
...
[10:14:29] Launching rppg_injector.py "...path with spaces..." --inject ...
[10:14:36] rPPG backend — CuPy unavailable (RuntimeError); frame math stays on CPU.   <- INJECTOR subprocess: CuPy BROKEN
[10:14:37] MediaPipe GPU delegate unavailable (NotImplementedError ...); using CPU.    <- separate issue (see A4)
```

**This is the diagnosis you must confirm and fix:** CuPy imports + runs the GPU
op fine in the **GUI's** Python process (so the wheels + nvrtc DLLs ARE installed
in his venv), but **fails with a bare `RuntimeError` inside the
`rppg_injector.py` SUBPROCESS**. The injector swallows the real exception detail
(`feedback`-worthy: it only logs "RuntimeError", not the message). The most
likely root causes, in order — VERIFY each, don't assume:

- **A1 — `os.add_dll_directory()` not running / running too late in the injector
  subprocess.** `rPPG/rppg_injector.py` registers the `site-packages/nvidia/*/bin`
  dirs before `import cupy` (the nvrtc fix; memory `project_v217_rppg_killer_matplotlib`
  + the rppg-wiring doc). In his patched v2.17 build this may be absent, mis-ordered,
  or the handles list got GC'd. The GUI works because it's a different entry
  point that may set the DLL dirs differently (or CuPy was already warm).
- **A2 — subprocess doesn't inherit the DLL search path / a needed env var.** The
  GUI launches `rppg_injector.py` as a child process; if the child's
  `os.add_dll_directory` block is guarded behind a condition that's false in the
  child, or PATH/CUDA_PATH isn't inherited, nvrtc won't load. NOTE his log:
  `CUDA path could not be detected. Set CUDA_PATH ...` warning appears on this
  box class. **TDD: spawn the injector as a real subprocess from a test and
  assert it logs GPU-ready, on a path WITH SPACES.**
- **A3 — the injector swallows the real error.** First fix (cheap, high-value):
  make `rppg_injector.py` log the FULL exception (`type + message + the missing
  DLL name`) instead of just `RuntimeError`. You cannot fix what you can't see —
  and the friend's next log will then tell you the exact missing DLL. This alone
  is worth shipping immediately.
- **A4 — MediaPipe GPU delegate "disabled in build flags" is SEPARATE and
  EXPECTED.** The pip mediapipe wheel is CPU-only (no GPU delegate); that
  `NotImplementedError` is normal and NOT the 20-min problem. The FaceLandmarker
  runs fine on CPU and is fast. **Do not chase the MediaPipe GPU delegate** — the
  rPPG frame-math CuPy path (butterfly of `cupyx.scipy.ndimage`) is what's slow on
  CPU. Confirm this scoping but don't rabbit-hole on mediapipe-GPU.
